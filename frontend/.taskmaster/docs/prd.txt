# 협업 플랫폼 프론트엔드 PRD (Product Requirements Document)

## 프로젝트 개요
일러스트레이터와 클라이언트 간의 원활한 커뮤니케이션을 위한 실시간 협업 웹 플랫폼의 프론트엔드 개발

## 디자인 요구사항
- Apple 스타일의 미니멀하고 모던한 블랙&화이트 디자인
- 깔끔하고 직관적인 UI/UX
- 반응형 디자인 (데스크톱 우선)
- 부드러운 애니메이션과 트랜지션
- 높은 가독성과 접근성

## 기술 스택
- Framework: Next.js 14+ (App Router)
- Language: TypeScript
- Styling: Tailwind CSS + shadcn/ui
- State Management: Zustand
- Form Handling: React Hook Form + Zod
- Image Handling: react-zoom-pan-pinch
- Drawing Tools: Konva.js 또는 Fabric.js
- Real-time: Socket.io-client (추후 백엔드 연동)

## 핵심 기능 (Phase 1 - 프론트엔드)

### 1. 인증 시스템 UI
- 로그인 페이지 (아이디/비밀번호 입력)
- 회원가입 페이지 (아이디, 이메일, 비밀번호, 닉네임)
- 세션 관리 UI
- 자동 로그인 체크박스

### 2. 스튜디오 대시보드
- 프로젝트 목록 그리드/리스트 뷰
- 프로젝트 카드 (썸네일, 제목, 상태, 마감일)
- 필터링 옵션 (일러스트/스토리보드)
- 검색 기능
- 알림 아이콘 및 배지

### 3. 프로젝트 관리 인터페이스
- 프로젝트 생성 모달
- 프로젝트 정보 입력 폼
- 초대 코드 생성 및 복사 UI
- 참여자 목록 표시
- 프로젝트 설정 페이지

### 4. 씬 에디터 뷰
- 씬 탭 네비게이션
- 이미지 업로드 영역 (드래그 앤 드롭)
- 선화/아트 구분 업로드 섹션
- 이미지 뷰어 (확대/축소/패닝)
- 버전 히스토리 사이드바
- 씬 설명 텍스트 에디터

### 5. 협업 도구 UI
- 댓글 섹션 (스레드 형식)
- 이미지 주석 도구바
- 드로잉 도구 패널
- 비교 모드 토글
- 실시간 커서 표시 (Mock)

### 6. 사용자 프로필
- 프로필 설정 페이지
- 아바타 업로드
- 정보 수정 폼

### 7. 관리자 패널 (Admin)
- 사용자 관리 테이블
- 프로젝트 관리 대시보드
- 시스템 통계 차트

## 개발 우선순위

### Phase 1 (1-2주차): 기초 설정 및 핵심 페이지
1. Next.js 프로젝트 초기 설정
2. 디자인 시스템 구축 (컬러, 타이포그래피, 컴포넌트)
3. 레이아웃 컴포넌트 (Header, Sidebar, Footer)
4. 인증 페이지 UI (로그인/회원가입)
5. 스튜디오 대시보드 페이지

### Phase 2 (3-4주차): 프로젝트 및 씬 관리
1. 프로젝트 생성/관리 UI
2. 씬 에디터 기본 구조
3. 이미지 업로드 컴포넌트
4. 이미지 뷰어 구현
5. 버전 관리 UI

### Phase 3 (5-6주차): 협업 기능
1. 댓글 시스템 UI
2. 주석 도구 구현
3. 드로잉 도구 통합
4. 비교 모드 구현

### Phase 4 (7-8주차): 최적화 및 통합
1. 상태 관리 최적화
2. Mock API 구현
3. 반응형 디자인 완성
4. 성능 최적화
5. 접근성 개선

## 성공 지표
- Lighthouse 성능 점수 90+ 
- 모든 주요 브라우저 호환성
- 3초 이내 초기 로딩
- 직관적인 사용자 경험
- 깔끔한 코드 구조

## 제약사항
- 백엔드 API 없이 Mock 데이터로 구동
- 실시간 기능은 시뮬레이션으로 구현
- 파일 업로드는 로컬 스토리지/IndexedDB 활용

## 참고사항
- Apple Human Interface Guidelines 참조
- 기존 Ignite 템플릿 구조 참고
- 모바일 우선이 아닌 데스크톱 우선 설계
- 다크모드는 추후 고려
